C



History

In the sixties, Dennis Ritchie along with other employees of Bell Labs (AT&T) worked on a project called Multics, which goal was to develop an operative system to run on a large computer and to be used for thousands of users simultaneously. Unfortunately, in 1969, the project was aborted due to high costs. So, they had to search for new projects. Ken Thompson, another worker of Bell Labs, develop a whole new file system in assembly language. After a while, also Dennis Ritchie started to make improvements on the system, and Unix was born. Unix later was improved on Fortran and B. B was written by Ken Thompson in 1970 and it is called the predecessor of C. Although it is a high-level language, B had some problems: it did not recognize data-types nor "structures". So, in 1971, Dennis Ritchie started to improve the B language, turning it into C. It keeped most of the B syntax, but added many other changes, like data-types. The C language was very powerful, yet high-level programming, making it the default language when programming operative systems and most of the software. Eventually, Unix ended up being completely rewritten in C.

"When I read commentary about suggestions for where C should go, I often think back and give thanks that it wasn't developed under the advice of a worldwide crowd."

Dennis Ritchie



Advantages

The language C has some characteristics that make it really powerful and a great language to programm with:

  • Low-level or high-level programming. C language provides low-level access to the memory, making it possible to run low-level activities. Besides, it has high-level constructors, making it easier for the user to write the program. Officially, C is a mid-level language.
  • Portability. Any C program is independent on any platform, does not depend on the configurations of the machine.
  • Modularity. This is one of the most appreciated features of C. We can "break" a program into small ones for making it easier to write and/or read. Also, the debugging will become faster if well written.
  • Speed. As a mid-level language, C is faster to compile than other high-level languages, like Java.
  • The language. C is a very structured language, with few reserved words, making it easier to the programmer to write programms. Due to the modularity, C is also extensible, because we can add what we want to any program without have to write many line of code, just calling the functions already created.

    • "[C has] the power of assembly language and the convenience of … assembly language."

    Dennis Ritchie



Data Types

In C, a data type is used to declare variables of different types. The type determines how much space it will occupy in the storage. There are a few types of Data:

  1. Basic Types. Simple arithmetic types. The types most used in a C program. There are some that occupy more memory than others, the most used are int (2 or 4 bytes), for integers, char (1 byte) for characters, and floating point to represent decimal places, like float (4 bytes).
  2. Boolean type. A boolean type ("bool") is really simple to use. It returns "true" or "false" and can only have these two values.
  3. Structures. Structures are a way to store dynamically different types of different data. A structure can have any of the other types of data inside it.
  4. Arrays. Arrays are ordered sequences of numbers or characters. If we want to print a sentence, we create a String. A string is a sub-type of an array, for it is a sequence of characters.
  5. Pointers. A pointer is a new way to deal with memory problems. It is a variable that stores an address of memory. For example, if we want to know the exact physical memory location of an integer, we have to use a pointer, pointing to that integer.



Memory Management

Like it was said before, although C is a medium-level language, has also the power to adress directly the memory adress of a computer, making it easier to create some specific programs. For example, we want to create an array, but we don't know how long it will be its size. Since we can't change the dimension after it has been initialized, how do we do? Dinamic memory allocation is a way to fix this problems. There are four functions to adress this topic: calloc, free, malloc and realloc.

The Malloc function provides a manual (by the programmer) memory allocation. Which means that malloc does not return a number, but a pointer to the memory created. After the initialization it is needed an error checking. It is possible that, if the memory that we want to initialize is more than the memory available, the pointer comes null. The difference between the malloc function and the calloc funtion is that the calloc function initializes the memory adresses to zero. The Realloc function resizes the memory passed by a pointer. Lastly, free releases the memory that was allocated by the last three functions. These four functions are very important in C when the programmer wants to otimize a software, rezising his memory in his course, allowing the program to be faster.



Uses of C

We have seen that C is a very powerfull language and one of the most used in the world. But why use C?

  1. Operating Systems. C was designed because there was a need to write Unix in a simple language, yet with funcionalities of low-level (memory allocation). So, C is the default language to write operating systems.
  2. Language compilers. C can also be used to create a language compiler of high level, almost as an intermediate between the machine and the language itself.
  3. Assemblers. C can be used as a language compiler, but also as an assembler, doing the same thing as a language compiler, but for one specific language (assembly).
  4. Text Editors. One of the main uses of C today is the creation of text editors, not only for programming, but just simple text editors for daily use.
  5. Print Spoolers. A print spooler is a software program responsible for managing all print jobs currently being sent to the computer printer or print server. The print spooler program may allow a user to delete a print job being processed or otherwise manage the print jobs currently waiting to be printed.
  6. Network Drivers. A network driver is a software responsible for connecting a computer to a network. Computers usually have ethernet and wireless: without network drivers, these devices do not work properly.
  7. Modern Programs. Almost all of the major modern programs can be written in C! In fact, most of them actually are.
  8. Data Bases. Data base creation can be made in C, due to C memory dinamic allocation, which makes it easier to create data bases.

Simple Program

Here is a simple C Hello World program:

        //C hello world example
        #include <stdio.h>

        int main()
        {
            printf("Hello world\n");
            return 0;
        }

The first line, that begins with "//", is a simple comment, that is ignored when compiling the code. The next line, "#include <stdio.h>", includes the library <stdio.h> (standard IO). Now, the code will be able to read or write lines form the keyboard, or to print them in the monitor.

Next to that, we have the main() function. Everything that the program will run after it has compiled is the main function. Then, we call the printf function, that prints the argument on the monitor. Lastly, the return determines what the function will return. In the main function, if the program has no bugs, it always returns 0.



C and derivatives

C has clearly influenced many languages, mostly syntactically. In 1979 C++ was created, by Bjarne Stroustrup, another Bell Labs employee. It is called the "C with Classes". It is today a big complement for C, because it is an object-oriented programming language. Although it does not offer the same memory allocation as C, C++ is called the "Super C". In 1986, Objective-C was created, an object-oriented dynamic language based heavily on Smalltalk and C. On the following years, it would emerge new object-oriented programming languages based in C that would became very popular, like Perl, Python, Java and C#.

"C++ and Java, say, are presumably growing faster than plain C, but I bet C will still be around."

Dennis Ritchie
